- Теперь в ответе включены товары для каждой продажи
- Поле `items` больше не `null`, а содержит массив товаров

## 🎯 Технические детали

### Оптимизация производительности:
1. **Один запрос для продаж** - загружаем все продажи с фильтрами
2. **Один запрос для товаров** - загружаем все товары для всех продаж одним запросом
3. **Группировка в памяти** - группируем товары по `sale_id`

### Поля в ответе:

#### Основные поля продажи:
- `id` - ID продажи
- `customer_id` - ID клиента
- `customer_name` - Имя клиента
- `total_amount` - Общая сумма
- `payment_status` - Статус оплаты (PAID, PARTIAL, DEBT)
- `stage` - Этап продажи (ordered, ready, delivered)
- `created_at` - Дата создания

#### Поля товаров (items):
- `id` - ID товара в продаже
- `sale_id` - ID продажи
- `product_id` - ID товара
- `product_name` - Название товара
- `product_code` - Код товара
- `stock_item_id` - ID партии (для batch товаров)
- `quantity` - Количество
- `unit_price` - Цена за единицу
- `unit_value` - Значение единицы (для рулонов и т.д.)
- `total_price` - Общая цена товара
- `style_id` - ID стиля (если есть)
- `style_name` - Название стиля (если есть)
- `status` - Статус записи

## 🔄 Фильтрация

Эндпоинт поддерживает те же фильтры, что и раньше:

### По дате:
```
GET /api/sales?date=2024-01-01
```

### По месяцу и году:
```
GET /api/sales?month=1&year=2024
```

### По году:
```
GET /api/sales?year=2024
```

## ⚡ Преимущества

1. **Улучшенный UX** - пользователи сразу видят состав продаж
2. **Меньше запросов** - не нужно делать отдельные запросы для товаров
3. **Оптимизировано** - товары загружаются одним запросом для всех продаж
4. **Совместимость** - формат ответа остался прежним, только `items` теперь заполнен

## 🚨 Важные моменты

1. **Обратная совместимость** - если у продажи нет товаров, `items` будет пустым массивом `[]`
2. **Производительность** - для большого количества продаж может быть медленнее, но оптимизировано
3. **Формат товаров** - такой же как в `GET /api/sales/:id`

## 🔄 Сравнение с другими эндпоинтами

### GET /api/sales (теперь):
- Возвращает продажи + товары
- Оптимально для списков

### GET /api/sales/:id (без изменений):
- Возвращает одну продажу + товары
- Оптимально для детальной страницы

### GET /api/reports/sales (без изменений):
- Возвращает отчет с товарами
- Оптимально для аналитики
