# Исправление расчета количества в продажах

## Проблема
При продаже товаров система некорректно рассчитывала остатки. Вместо реального количества (`quantity * unit_value`) использовалось только `quantity`.

## Пример проблемы

**До исправления:**
```json
{
  "product_id": 107,
  "quantity": 1,
  "unit_value": 3,
  "unit_price": "25.00"
}
```
- Списывалось со склада: 1 единица ❌
- Должно списываться: 1 × 3 = 3 единицы ✅

## Что исправлено

### 1. Метод `create` (создание продажи)
```javascript
// Было:
await connection.execute(
  'UPDATE stock SET quantity = quantity - ? WHERE product_id = ?',
  [item.quantity, item.product_id]
);

// Стало:
const realQuantity = item.quantity * (item.unit_value || 1);
await connection.execute(
  'UPDATE stock SET quantity = quantity - ? WHERE product_id = ?',
  [realQuantity, item.product_id]
);
```

### 2. Метод `update` (обновление продажи)
- Исправлено списание остатков для новых товаров
- Исправлено восстановление остатков для старых товаров

### 3. Метод `remove` (удаление продажи)
- Исправлено восстановление остатков при удалении
- Добавлено поле `unit_value` в запрос

## Логика расчета

### Реальное количество товара:
```
real_quantity = quantity × unit_value
```

### Примеры:
- 1 коробка × 12 штук = 12 единиц
- 2 упаковки × 5 штук = 10 единиц
- 3 пачки × 1 штука = 3 единицы

### Для партионных товаров:
- Обновляется общий остаток в `stock`
- Обновляется остаток конкретной партии в `stock_items`
- Партия деактивируется при достижении нулевого остатка

## Обновленные запросы

### Списание остатков:
```sql
UPDATE stock SET quantity = quantity - ? WHERE product_id = ?
```
Где `?` = `quantity * unit_value`

### Восстановление остатков:
```sql
UPDATE stock SET quantity = quantity + ? WHERE product_id = ?
```
Где `?` = `quantity * unit_value`

### Для партий:
```sql
UPDATE stock_items SET quantity = quantity - ? WHERE id = ? AND product_id = ?
```

## Тестирование

### Тест 1: Простая продажа
```json
{
  "product_id": 107,
  "quantity": 1,
  "unit_value": 3,
  "unit_price": "25.00"
}
```
**Ожидаемый результат:** Остаток уменьшится на 3 единицы

### Тест 2: Несколько товаров
```json
{
  "items": [
    {
      "product_id": 5,
      "quantity": 2,
      "unit_value": 5,
      "unit_price": 2
    },
    {
      "product_id": 5,
      "quantity": 3,
      "unit_value": 2,
      "unit_price": 2
    }
  ]
}
```
**Ожидаемый результат:** 
- Товар 5: общее списание = (2×5) + (3×2) = 16 единиц
- Общая сумма = (2×5×2) + (3×2×2) = 32

### Тест 3: Отмена продажи
При отмене продажи должно восстановиться корректное количество единиц.

## Важные моменты

1. **Обратная совместимость:** Существующие продажи в базе данных останутся корректными
2. **Партионные товары:** Корректно работает с партиями
3. **Защита от ошибок:** `unit_value || 1` защищает от null значений
4. **Транзакции:** Все изменения в рамках одной транзакции
