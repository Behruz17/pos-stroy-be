# API Документация: Товары и Партии

## Эндпоинт: GET /api/products

Возвращает список товаров с партиями. Каждая партия отображается как отдельная запись.

### Формат ответа:

#### Для партий товара:
```json
{
  "id": 1,                    // ID партии (stock_item_id)
  "product_id": 123,          // ID товара
  "name": "ТоварА",           // Название товара
  "manufacturer": "Производитель",
  "created_at": "2023-01-01T00:00:00.000Z",
  "image": "/uploads/products/image.jpg",
  "notification_threshold": 10,
  "product_code": "CODE001",
  "type": "batch",
  "status": 1,
  "batch_code": "BATCH001",   // Код партии
  "stock_quantity": 10,       // Количество в этой партии
  "purchase_cost": 100,       // Цена закупки партии
  "selling_price": 150,       // Цена продажи партии
  "currency": "TJS",
  "rate": 1.0000,
  "purchase_cost_converted": null,
  "receipt_id": 456,          // ID прихода
  "is_batch_item": true       // 🔑 Флаг для определения партии
}
```

#### Для товаров без партий:
```json
{
  "id": 123,                  // ID товара
  "product_id": 123,          // ID товара
  "name": "ТоварБезПартии",
  "manufacturer": "Производитель",
  "created_at": "2023-01-01T00:00:00.000Z",
  "image": "/uploads/products/image.jpg",
  "notification_threshold": 10,
  "product_code": "CODE002",
  "type": "simple",
  "status": 1,
  "batch_code": null,         // Нет партии
  "stock_quantity": 0,        // Нулевой остаток
  "purchase_cost": null,
  "selling_price": null,
  "currency": null,
  "rate": null,
  "purchase_cost_converted": null,
  "receipt_id": null,
  "is_batch_item": false      // 🔑 Флаг для определения товара
}
```

## 🎯 Требования для фронтенда

### 1. Отображение в списке:
- Каждая партия отображается как отдельная строка
- Название товара одинаковое для всех партий одного товара
- Показывать `batch_code` для идентификации партии
- Показывать `stock_quantity` для каждой партии отдельно

### 2. Кнопки действий:
```javascript
// Для товаров (is_batch_item: false)
if (!item.is_batch_item) {
  // Показать кнопки:
  // - "Редактировать товар"
  // - "Удалить товар"
}

// Для партий (is_batch_item: true)
if (item.is_batch_item) {
  // Скрыть кнопки редактирования/удаления товара
  // Можно добавить специфичные для партий кнопки:
  // - "Детали партии"
  // - "История продаж партии"
  // - "Корректировка остатка"
}
```

### 3. Сортировка:
- Сортировка по названию товара (`name`)
- Затем по дате создания партии (`created_at`)

### 4. Фильтрация и поиск:
- Поиск должен работать по названию товара
- Фильтрация по производителю
- Фильтрация по типу товара (`type`)

### 5. Визуальное различие:
- Партии можно выделить другим цветом или иконкой
- Показывать `batch_code` prominently
- Добавить индикатор что это "партия" а не "товар"

### 6. Обработка ошибок:
- Если `stock_quantity` = 0, показать соответствующий индикатор
- Если `batch_code` = null, это товар без партий

## 🔄 Изменения в логике

### Раньше:
```
ТоварА - остаток 50 (сумма всех партий)
```

### Теперь:
```
ТоварА (BATCH001) - остаток 10
ТоварА (BATCH002) - остаток 15  
ТоварА (BATCH003) - остаток 25
```

## ⚠️ Важные моменты

1. **ID разница:** Для партий `id` = `stock_item_id`, для товаров `id` = `product_id`
2. **Уникальность:** Каждая партия уникальна по `batch_code`
3. **Цены:** У каждой партии свои цены закупки/продажи
4. **Валюта:** Каждая партия может иметь свою валюту и курс
5. **purchase_cost_converted:** Всегда `null` для партий (поле не реализовано в stock_items)

## 🚀 Дополнительные эндпоинты

- `GET /api/products/:id/stock-items` - детальная информация о партиях конкретного товара
- `GET /api/products/:id` - информация о конкретном товаре (старый формат)
