# API Документация: Товары

## Эндпоинт: GET /api/products

Возвращает список товаров с агрегированными данными об остатках и ценах.

### Формат ответа:

```json
{
  "id": 123,                        // ID товара
  "name": "ТоварА",                 // Название товара
  "manufacturer": "Производитель",
  "created_at": "2023-01-01T00:00:00.000Z",
  "image": "/uploads/products/image.jpg",
  "notification_threshold": 10,
  "product_code": "CODE001",
  "type": "batch",                  // "batch" или "simple"
  "status": 1,
  "stock_quantity": 50,             // Общий остаток товара
  "purchase_cost": 100,             // Цена закупки из последней партии
  "selling_price": 150,             // Цена продажи из последней партии
  "purchase_cost_converted": 100,   // Сконвертированная цена закупки
  "currency": "TJS",                // Валюта последней партии
  "rate": 1.0000                    // Курс валюты
}
```

## 🎯 Требования для фронтенда

### 1. Отображение в списке:
- Каждый товар отображается как одна строка
- `stock_quantity` показывает общий остаток всех партий
- Для товаров типа "batch" можно показать детализацию по партиям через отдельный эндпоинт

### 2. Кнопки действий:
```javascript
// Для всех товаров
{
  // - "Редактировать товар"
  // - "Удалить товар"
  // - "Просмотреть партии" (только для type="batch")
}
```

### 3. Сортировка:
- Сортировка по ID товара (`id`)

### 4. Фильтрация и поиск:
- Поиск по названию товара (`name`)
- Фильтрация по производителю (`manufacturer`)
- Фильтрация по типу товара (`type`)

### 5. Визуальное различие:
- Показывать тип товара (`type`) - "batch" или "simple"
- Для "batch" товаров можно добавить иконку или индикатор

### 6. Обработка ошибок:
- Если `stock_quantity` = 0, показать соответствующий индикатор
- Если `purchase_cost` = null, значит товар еще не поступал на склад

## 🔄 Логика работы

### Остатки:
- `stock_quantity` - это суммарный остаток всех партий товара
- Для простых товаров (`type="simple"`) это прямой остаток из таблицы `stock`

### Цены:
- `purchase_cost` и `selling_price` берутся из последней поступившей партии
- `purchase_cost_converted` - цена в базовой валюте с учетом курса

## ⚠️ Важные моменты

1. **Уникальность:** Каждый товар уникален по `product_code`
2. **Типы товаров:** `simple` - обычные товары, `batch` - товары с партионным учетом
3. **Агрегация:** Данные цен берутся из последней партии, остатки суммируются
4. **Валюта:** Показывается валюта последней поступившей партии

## 🚀 Дополнительные эндпоинты

- `GET /api/products/:id/stock-items` - детальная информация о партиях конкретного товара
- `GET /api/products/:id` - информация о конкретном товаре (старый формат)
