# Автоматическое создание партий для партионных товаров

## Обзор
Добавлена функция автоматического создания партии для партионных товаров (`type='batch'`) при продаже, если для товара еще не созданы партии в `stock_items`.

## Логика работы

### При создании продажи (create) и обновлении (update):

1. **Проверка типа товара**: Система проверяет `type` товара
2. **Для партионных товаров**:
   - Проверяет наличие существующих партий в `stock_items`
   - **Если партий нет**: автоматически создает одну партию с текущим остатком из `stock`
   - **Если партии есть**: требует указания `stock_item_id`

### Процесс автоматического создания партии:

1. **Получение текущего остатка** из таблицы `stock`
2. **Создание партии** в `stock_items` с параметрами:
   - `product_id`: ID товара
   - `quantity`: текущий остаток из `stock`
   - `batch_code`: `AUTO-BATCH-{product_id}-{timestamp}`
   - `purchase_cost`: `null`
   - `selling_price`: `null`
   - `status`: `1` (активна)

3. **Использование созданной партии** для текущей продажи

## Примеры использования

### Сценарий 1: Первая продажа партионного товара
```json
{
  "items": [
    {
      "product_id": 123,
      "quantity": 5,
      "unit_price": 1000
    }
  ]
}
```

**Результат:**
- Автоматически создана партия с кодом `AUTO-BATCH-123-1648123456789`
- Партия содержит весь текущий остаток товара
- Продажа использует созданную партию

### Сценарий 2: Продажа с существующими партиями
```json
{
  "items": [
    {
      "product_id": 123,
      "stock_item_id": 456,
      "quantity": 3,
      "unit_price": 1000
    }
  ]
}
```

**Результат:**
- Используется указанная партия `stock_item_id: 456`
- Проверяется достаточность количества в партии

### Сценарий 3: Попытка продажи без указания партии (когда партии существуют)
```json
{
  "items": [
    {
      "product_id": 123,
      "quantity": 2,
      "unit_price": 1000
    }
  ]
}
```

**Результат (ошибка):**
```
"Stock item (batch) is required for batch product 123. Available batches: 3"
```

## Преимущества

1. **Автоматизация**: Не нужно вручную создавать партии для новых партионных товаров
2. **Гибкость**: Можно указывать конкретную партию, если нужно
3. **Обратная совместимость**: Существующая логика с указанием `stock_item_id` работает как раньше
4. **Удобство**: Идеально для розничных продаж, где не важна конкретная партия

## Важные моменты

1. **Только для партионных товаров**: Автоматическое создание работает только для товаров с `type='batch'`
2. **Одна партия**: Создается только одна партия со всем остатком
3. **Уникальный код**: Каждая автоматическая партия получает уникальный `batch_code`
4. **Без цен**: Автоматические партии создаются без `purchase_cost` и `selling_price`

### Добавленная логика:
```javascript
// Проверка существующих партий
const [existingBatches] = await connection.execute(
  'SELECT id, quantity FROM stock_items WHERE product_id = ? AND status = 1',
  [item.product_id]
);

// Автоматическое создание партии при необходимости
if (existingBatches.length === 0) {
  const batchCode = `AUTO-BATCH-${item.product_id}-${Date.now()}`;
  const [batchResult] = await connection.execute(
    'INSERT INTO stock_items (product_id, quantity, batch_code, purchase_cost, selling_price, status) VALUES (?, ?, ?, ?, ?, ?)',
    [item.product_id, currentStock, batchCode, null, null, 1]
  );
  stockItemId = batchResult.insertId;
}
```
