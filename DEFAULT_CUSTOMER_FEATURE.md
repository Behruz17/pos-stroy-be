# Клиент по умолчанию для розничных продаж

## Обзор
Добавлена возможность назначить одного клиента как "клиент по умолчанию" для розничных продаж. Если при создании продажи не указан клиент, автоматически подставится клиент по умолчанию.

## Что было сделано

### 1. Изменения в базе данных
- Добавлено поле `is_default` в таблицу `customers`
- Создан индекс для быстрого поиска клиента по умолчанию

### 2. Обновления в коде

#### Модуль продаж (`sales.service.js`)
- Добавлена логика автоматической подстановки клиента по умолчанию, если `customer_id` не указан
- При создании продажи система ищет клиента с флагом `is_default = 1`

#### Модуль клиентов (`customer.service.js`)
- `getDefaultCustomer()` - получение клиента по умолчанию
- `setDefaultCustomer(id)` - установка клиента по умолчанию (сбрасывает все остальные флаги)

#### Новые API эндпоинты
- `GET /api/customers/default/current` - получить текущего клиента по умолчанию
- `PUT /api/customers/{id}/default` - установить клиента как клиент по умолчанию

## Установка

### 1. Выполните миграцию базы данных
```sql
-- Выполните SQL из файла setup_default_customer.sql
ALTER TABLE customers ADD COLUMN is_default TINYINT(1) DEFAULT 0;
CREATE INDEX idx_customers_default ON customers(is_default, status);
```

### 2. Назначьте клиента по умолчанию
Через API:
```bash
PUT /api/customers/{customer_id}/default
```

Или напрямую в базе данных:
```sql
UPDATE customers SET is_default = 1 WHERE id = {customer_id} AND status = 1;
```

## Использование

### Создание продажи без клиента
Теперь вы можете создать продажу без указания `customer_id`:

```json
{
  "payment_status": "PAID",
  "cash_amount": 1000,
  "items": [
    {
      "product_id": 1,
      "quantity": 2,
      "unit_price": 500
    }
  ]
}
```

Если существует клиент по умолчанию, он автоматически будет назначен на эту продажу.

### Создание продажи с конкретным клиентом
Если вы указываете `customer_id`, будет использован указанный клиент (как и раньше):

```json
{
  "customer_id": 123,
  "payment_status": "PAID", 
  "cash_amount": 1000,
  "items": [...]
}
```

## Важные моменты

1. **Только один клиент по умолчанию**: Система гарантирует, что только один клиент может иметь флаг `is_default = 1`
2. **Автоматическая подстановка**: Происходит только если `customer_id` не указан или равен `null`
3. **Обратная совместимость**: Существующий код продолжит работать без изменений
4. **Розничные продажи**: Идеально подходит для розничных продаж, где не важен конкретный клиент

## API Примеры

### Получить клиента по умолчанию
```bash
GET /api/customers/default/current
```

### Установить клиента по умолчанию
```bash
PUT /api/customers/72/default
```

### Создание розничной продажи (без клиента)
```bash
POST /api/sales
{
  "payment_status": "PAID",
  "cash_amount": 1500,
  "items": [
    {
      "product_id": 1,
      "quantity": 3,
      "unit_price": 500
    }
  ]
}
```
